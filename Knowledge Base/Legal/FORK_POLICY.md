# CoSyn Fork Policy

Version 1.0

## Purpose

This policy clarifies how forks of the CoSyn repository may be created and distributed while protecting the integrity of the original project.

## Open Source Rights

The repository license permits users to:

- fork the repository
- modify the code
- experiment with governance mechanisms
- develop derivative implementations

These rights are subject to the terms of the repository license.

## Naming Restrictions

Forked projects must clearly distinguish themselves from the official CoSyn project.

Forks may not:

- use the name "CoSyn" as the primary product name
- imply official endorsement
- present themselves as official releases

Examples:

Allowed:

- "AI Governance Lab — based on CoSyn"
- "Project Aurora (inspired by CoSyn)"

Not allowed:

- "CoSyn Enterprise"
- "Official CoSyn Fork"
- "CoSyn Pro"

## Trademark Protection

The name **CoSyn** and associated marks may be protected under trademark law.

Forks must not use these marks in ways that create user confusion.

## Distribution of Forks

Fork maintainers are responsible for:

- clearly identifying their project as a fork
- documenting modifications
- maintaining their own support channels

## Official Releases

Only releases published by the original repository maintainers should be considered official CoSyn releases.

## Enforcement

The project maintainers reserve the right to request changes to forks that create brand confusion or violate trademark rights.
